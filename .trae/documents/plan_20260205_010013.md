# Implementation Plan: Backend & Database Integration for Implementation Files

## 1. Database Schema & Data Initialization
Create a new table `t_implementation_plan_files` to store file records and populate it with mock data linking to existing static files.

### SQL Script (`init_files.sql`)
- **Table Structure**:
  - `id`: Primary Key
  - `category`: 'plan_text' (方案文本), 'charts' (重要图表), 'materials' (支撑材料)
  - `file_type`: '文本文件', '图表文件', etc.
  - `file_name`: Display name (e.g., '基础条件.pdf')
  - `file_path`: Relative path to static uploads
  - `upload_time`: Timestamp
  - `status`: 'success' or 'fail'
- **Mock Data**: Insert records mapping to actual files found in `vue/public/static/uploads/`.

## 2. Frontend Integration (Vue Component)
Modify `src/views/async/implementation/basicProjectOverview/index.vue` to fetch data from the backend instead of using hardcoded lists.

### Data Fetching Logic
- Implement `fetchImplementationFiles()` using `publicStore.http`.
- Query `model: 't_implementation_plan_files'`.
- Distribute results into `state.draft.step2.fileList`, `chartsList`, and `materialsList` based on the `category` field.

### CRUD Operations (Query Interfaces)
Implement the following methods using the Project's "Data Middle Platform" patterns:
1.  **View File List**: `publicStore.http({ Api: { model: 't_implementation_plan_files' ... } })`
2.  **Upload File (Mock)**: Use `api.addApi` to insert a record into `t_implementation_plan_files`.
3.  **Delete File**: Use `api.deleteApi` (or generic update to mark deleted) if available, or simulated via state update for this task if strict backend API isn't fully exposed for deletes (will assume standard `deleteApi` or similar exists, otherwise will note it).
4.  **Check Status**: Re-query the specific file record by ID.
5.  **View File**: Generate the download/preview URL using the `file_path`.

## 3. Execution Steps
1.  Create the SQL script file `init_files.sql` for the user.
2.  Modify `index.vue` to replace static data initialization with the API call.
3.  Add the `fetchImplementationFiles` function and call it on mount/init.
4.  Add the requested "Query Interface" methods (Upload, Delete, View, Status) as code blocks or directly into the component.
